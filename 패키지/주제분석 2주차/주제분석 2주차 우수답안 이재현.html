우선, 상관계수 플랏을 보자마자 느낄 수 있는 것은 일부를 제외한(apt와 동) 변수들 간에는 상관관계가 거의 존재하지 않는다고 보는게 타당하다는 것이다. Ridge 회귀의 식을 보면 회귀계수의 제곱에 alpha 값으로 제약을 주는 모양이다. 이는 가중치의 절대값을 작게 해서 과대 적합이 되지 않게 제약을 거는 것인데, 이러한 특성은 결국 변수 간 상관관계가 높은 모델에서 더 잘 예측하는 결과로 이어진다. 정리하자면 Ridge 회귀는 변수간 상관관계가 높은 상황에서 Lasso 회귀보다 좋은 성능을 내는 게 장점이지만, 위와 같은 상황에서는 변수간 상관관계가 매우 낮으므로 적절하다고 생각되지 않는다.

def plot_predictions(name_, pred, actual):
    df = pd.DataFrame({'prediction': pred, 'actual': actual})
    df = df.sort_values(by='actual').reset_index(drop=True)

    plt.figure(figsize=(8, 7))
    plt.scatter(df.index, df['prediction'], marker='x', color='r')
    plt.scatter(df.index, df['actual'], alpha=0.7, marker='o', color='black')
    plt.title(name_, fontsize=15)
    plt.legend(['prediction', 'actual'], fontsize=12)
    plt.show()
    
def mse_eval(name_, pred, actual):
    global my_predictions, colors, my_pred, my_actual, my_name
    
    colors = ['r', 'c', 'm', 'y', 'k', 'khaki', 'teal', 'orchid', 'sandybrown',
          'greenyellow', 'dodgerblue', 'deepskyblue', 'rosybrown', 'firebrick',
          'deeppink', 'crimson', 'salmon', 'darkred', 'olivedrab', 'olive', 
          'forestgreen', 'royalblue', 'indigo', 'navy', 'mediumpurple', 'chocolate',
          'gold', 'darkorange', 'seagreen', 'turquoise', 'steelblue', 'slategray', 
          'peru', 'midnightblue', 'slateblue', 'dimgray', 'cadetblue', 'tomato'
         ]
    
    my_predictions = {}
    my_name = name_
    my_pred = pred
    my_actual = actual

    plot_predictions(name_, pred, actual)

    mse = mean_squared_error(pred, actual)
    my_predictions[name_] = mse

    y_value = sorted(my_predictions.items(), key=lambda x: x[1], reverse=True)
    
    df = pd.DataFrame(y_value, columns=['model', 'mse'])
    print(df)
    min_ = df['mse'].min() - 10
    max_ = df['mse'].max() + 10
    
    length = len(df) / 2
    
    plt.figure(figsize=(9, length))
    ax = plt.subplot()
    ax.set_yticks(np.arange(len(df)))
    ax.set_yticklabels(df['model'], fontsize=12)
    bars = ax.barh(np.arange(len(df)), df['mse'], height=0.3)
    
    for i, v in enumerate(df['mse']):
        idx = np.random.choice(len(colors))
        bars[i].set_color(colors[idx])
        ax.text(v + 2, i, str(round(v, 3)), color='k', fontsize=12, fontweight='bold', verticalalignment='center')
        
    plt.title('MSE Error', fontsize=16)
    plt.xlim(min_, max_)

2.4 해석¶

튜닝이 잘 안된 것 같다. 파라미터에 따른 값 차이가 거의 존재하지 않는다.

그 이유는 ridge 회귀의 수식에서 볼 수 있듯이 ridge의 alpha는 베타계수에 제약을 거는 역할을 하며, 이는 상관관계가 다소 높은 모델에서 효율적인 튜닝을 보여준다. 하지만 우리 모델은 위에서 확인한 바와 같이 상관관계가 적은 모델이므로 다른 회귀에 비해 성능이 떨어진다고 평가할 수 있다.

또한, ridge의 alpha 값이 베타계수에 제한(제약)을 걸 때, x값의 단위가 높으면 이는 제한을 걸어서 그 값을 낮추기 위해 매우 작은 alpha값을 필요로 하게 되고, alpha값이 작아짐에 따라 성능의 효율이 감소하게 된다. 예를 들어 alpha가 극단적으로 0으로 가게되면, 이는 일반 선형 회귀랑 다를게 없어진다는 것을 생각했을 때 납득하기 쉬울 것 같다.

3. LightGBM¶

3.1 LightGBM 이해¶

LGBM의 특징 : 일반 GBM 계열의 트리 분할 방법과 다르게 리프 중심 트리 분할 방식을 사용한다. 이 방식은 트리의 균형을 맞추지는 않고, 최대 손실 값을 가지는 리프 노드를 지속적으로 분할하면서 트리의 깊이가 깊어지고 비대칭적인 규칙 트리가 생성된다. 이렇게 최대 손실값을 가지는 리프 노드를 지속적으로 분할해 생성된 규칙 트리는 학습을 반복할수록 결국 균형 트리 분할 방식보다 예측 오류 손실을 최소화 할 수 있다고 생각한 것이 LGBM의 구현 사상이다.

장점 : 학습에 걸리는 시간이 적은 편이고, 메모리 사용량도 적은 편이다. 그러면서도 예측 성능이 XGBoost와 크게 차이나지 않는다. 오히려 기능상 다양성은 LGBM이 더 많은 편이다. 이는 LGBM이 XGBoost의 단점을 보완하고 장점을 계승해서 개발되었기 때문이다.

문제점 : 적은 데이터 세트를 적용할 경우 과적합이 발생하기 쉽다.

3.4 튜닝에 대해 알아보기¶

Grid Search : 하이퍼 파라미터의 후보군들을 직접 선택해서 집어넣는다. 장점으로는 사용자가 지정한 파라미터들의 결과를 알려준다는 점과 나름 균등하고 전역적 탐색이 가능하다는 점이 있지만, 단점으로는 파라미터 수가 증가하고 후보군의 개수가 증가하면 전체 탐색기간이 매우 길어진다는 점이 있다.

Random Search : 하이퍼 파라미터의 값의 범위를 선택하면, 그 안에서 임의의 수를 선택해서 탐색한다. 범위 안에서 무작위로 조합을 만들어서 에러가 작은 쪽으로 찾아가는 방식이다. 장점으로는 시간이 절약되어서 그리드 서치보다 파라미터 수가 많을 때 이용한다.

Baysian Optimization : 사전에 선택된 파라미터를 반영하여 탐색한다. 매 시도에 새로운 파라미터 값에 대한 조사 수행을 할 때 사전지식을 충분히 반영하면서 전체적인 탐색 과정을 체계적으로 수행하기 위해 고려해볼 수 있는 방법이다. 따라서, 장점으로는 Grid Search와 Random Search이 사전지식을 고려하지 않는 다는 점에 비해 이 방법은 사전지식을 고려한다는 점이 있다.
